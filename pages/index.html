QUESTION 1

Talk about how you made your site and why you chose the tools you did. Briefly explain one challenge you experienced in setting up this site and how you solved it.

After careful consideration, I chose Cactus as my site generator because I am already familiar with Python and that is the basis upon which Cactus is built. Once I'd made my choice, I installed Python, Cactus, and Git, and then created a new repo on Github. From there I cloned the repo locally so I could make the necessary changes to deploy on Netlify - specifically, the changes outlined under the "Preparing for Deployment" heading in this post from the Netlify blog. The challenge I faced while working through this process was working with Cactus inside a virtual environment. The first few resources I found were Mac/UNIX-specific guides and my personal machine runs Windows, and so I had to do a bit of Googling and digging on the Netlify blog and Support Forums to find what I needed to get things running.

QUESTION 5

Why do you think SSL/HTTPS is important? Please provide at least 3 reasons.

  1. HTTPS encrypts communications passed between a user and a site or service, making it more difficult for malicious actors to eavesdrop.
  2. SSL certificates confirm and validate the identity of a particular site or service. This makes it more difficult for malicious actors to impersonate a legitimate service.
  3. HTTPS can be faster than HTTP which means better accessibility for those on slower/limited network connections in addition to the security benefits.

QUESTION 6

Explain, in a couple of paragraphs, what you think 2 major challenges around DNS configuration are for less-technical customers hosting websites.

One of the big challenges a less-technical person might face with configuring DNS is understanding different network protocols and how to set them correctly on the server. For instance, if TCP/IP isn't configured correctly in the server OS, nothing will work as expected and a less-technical person might have a lot of difficulty diagnosing what's wrong.

Another challenge they might face is with maintenance and changes to an existing DNS configuration. There are guides which step through configuring a DNS server initially, but if changes need to be made (eg. adding a new network interface and pointing it to the correct DNS server) a less-technical person might run into a pitfall that a more technically proficient person would be able to avoid.

QUESTION 8

How would you set up an http 301 status redirect from “/netlify/anything” on your site, to https://www.google.com/search?q=anything? How about a proxy redirect? Please add that proxy redirect rule to your site.

To set up a redirect, I would edit the netlify.toml file and add the correct parameters as outlined in this Help Center article. Specifically, the "from" parameter would be set to "/netlify/anything" and the "to" parameter set to the Google search URL.

For the proxy redirect, I would follow the same steps but add the "status" parameter and set it to "200".

QUESTION 10

We understand you don't know anything about our internal procedures at this stage, but we want you to explain at a high level how you'd react to this situation: You receive a report of a severe security issue on www.netlify.com. What might you say to the reporter, even though we haven't confirmed their assertion yet, that will still leave the true impression that our business is very concerned about security? First, I would reply back to the customer to confirm that we've received the report - something to this effect:

Thanks for reporting this!

Security is extremely important to us - we seriously and thoroughly investigate the security vulnerability reports we receive. I'm sending this to our Security team for review along with the details you've provided, and I would also recommend filing a report through our Bug Bounty program if you haven't already done so (details here). We appreciate all of the members of the security community who collaborate with us to fix vulnerabilities, so thank you! Please let me know if you have any other concerns or questions and I'd be happy to help however I can.

I would then use the information provided in the report to validate the issue to the best of my ability, suggest a priority/severity level, and escalate to the Security team. If it was a very large and impactful problem, I would also alert the relevant teams and stakeholders via on-call/after-hours escalation to ensure the issue was addressed quickly.
